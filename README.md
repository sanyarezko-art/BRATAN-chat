# BRATAN-chat 🤝

Приватный P2P-чат с end-to-end шифрованием. **Без серверов. Без аккаунтов. Без истории.**
Полностью статический сайт на GitHub Pages.

👉 **Живой деплой:** <https://sanyarezko-art.github.io/BRATAN-chat/>

## Как это работает

- Ты создаёшь комнату. В браузере генерируется **32-байтный случайный секрет**.
- Секрет кладётся во фрагмент URL (`https://.../#<base64>`). Фрагмент по
  RFC 3986 **никогда не уходит ни на один сервер**, так что секрет остаётся
  у тебя в браузере.
- Из секрета локально через SHA-256 выводятся два значения:
  - `roomId` — его [Trystero](https://github.com/trystero-p2p/trystero)
    использует, чтобы найти других пиров через публичные BitTorrent-трекеры.
  - `password` — отдаётся в Trystero, тот делает PBKDF2 + AES-GCM на каждое
    сообщение поверх уже зашифрованного WebRTC data channel.
- Пиры соединяются **напрямую друг с другом** через WebRTC (DTLS-SRTP).
  Трекеры не видят содержимое — только SHA-256-хэш `roomId`.
- Кидаешь ссылку-приглашение тем, кого хочешь видеть в чате. У кого ссылка —
  тот внутри. У кого нет — тот снаружи.

## Модель угроз (честно)

**Что защищено:** содержимое сообщений в полёте, состав участников
(трекеры видят только хэш), история при хранении (её нет — ничего не
персистится вне вкладки).

**Что НЕ защищено:**

- У кого ссылка — тот может читать и писать. Относись к ссылке как к паролю.
  Новый разговор = новая комната + новая ссылка.
- Ники — самоназвания. Любой в комнате может представиться кем угодно.
  Не аутентифицируй по нику.
- Публичные BT-трекеры видят твой IP и roomId-хэш. Если важна приватность
  IP — VPN.
- Закрыл вкладку — сессия кончилась. «Прочитать позже» нет.

## Локально

Статический сайт, подойдёт любой HTTP-сервер:

```sh
python3 -m http.server 8080
# открой http://localhost:8080/
```

Trystero подгружается с [esm.sh](https://esm.sh/) — билд-шагов нет.
